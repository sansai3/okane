# メール処理アプリ(試作版)

このプロジェクトは、Python バックエンドと JavaScript フロントエンドで動くデスクトップアプリ(後で EXE 化予定)の試作です。まずはメニュー画面とメール処理、ZIP 展開処理を実装しています。

## 機能概要

- `config/app_config.json` に記載した6つのボタンがメニューとして表示されます。
- 設定ファイルを書き換えると、ボタンのタイトルや説明、呼び出す Python モジュールを自由に変更できます。
- 1つ目の「メール添付ダウンロード」ボタンでは、メールファイル(.eml/.msg)から添付とパスワード情報を整理します。
- 2つ目の「添付ファイル展開」ボタンでは、CSV に記録したパスワードを使って ZIP ファイルを一括展開します。
- 3つ目の「PDF変換」ボタンでは、選んだライブラリで PDF からテキストを抽出し、テキストまたは Word 形式で保存します。

## セットアップ

1. 依存パッケージをインストールします。
   ```bash
   pip install -r requirements.txt
   ```
2. 開発用サーバーを起動します。
   ```bash
   python app.py
   ```
3. ブラウザで `http://127.0.0.1:5000` を開きます。

## メール処理の流れ

1. メニューで「メール添付ダウンロード」を選ぶとフォームが開きます。
2. メールファイルが入っているフォルダ、添付を保存するフォルダ、CSVの保存先を入力します。
   - 何も入力しない場合は `config/app_config.json` の `paths` にある既定値が使われます。
3. 実行すると、以下を行います。
   - `.eml`/`.msg` ファイルを読み込み、添付ファイルを `001_ファイル名` のように連番で保存。
   - 添付が無いメールのみを対象に、本文からパスワード候補を抽出。
   - 英数字・記号の組み合わせ(全角/半角)や大小混在文字を優先し、説明文("information" など)を誤検出しないように調整。
   - 受信日/送信者/受信者/本文/パスワードを `data/mail_summary.csv` に書き出し。

## 圧縮ファイル展開の流れ

1. メニューで「添付ファイル展開」を選ぶとフォームが開きます。
2. 展開したい圧縮ファイルをまとめたフォルダ、パスワードを記録した CSV、展開後の保存先を入力します。
   - 何も入力しない場合は `paths.default_archive_folder` などの既定値が使われます。
3. 実行すると、以下を行います。
   - ZIP ファイルを順番に解凍し、失敗したものは `failed` フォルダへ移動。
   - パスワードが必要な場合は CSV の E 列に記録された値を総当たりで使用。
   - 展開後に現れた ZIP も再帰的に解凍し、日本語ファイル名の文字化けを防ぎながら保存。
   - 解凍できたアーカイブは `success` フォルダに移動し、内容は展開先にまとめます。

## PDF変換の流れ

1. メニューで「PDF変換」を選ぶとフォームが開きます。
2. 変換後のファイル形式(テキスト or Word)と使用するライブラリ(pdfplumber / pdfminer.six / PyMuPDF / 自動)を選びます。
   - 自動を選ぶと、設定ファイルの `pdf_processing.auto_engine_order` に書かれた順番でライブラリを試します。
3. PDF のあるフォルダと保存先フォルダを指定して実行すると、全PDFを UTF-8 (BOM 付き) のテキストまたは `.docx` として保存します。
   - 日本語や多言語の文字化けを防ぐため、テキストは UTF-8(BOM) で、Word は python-docx を使って出力します。

## 設定のカスタマイズ

- `paths.default_mail_folder` などを変更すると、フォームの初期値が変わります。
- `email_processing.password_keywords` に探したいキーワード(日本語/英語)を追加できます。
- `email_processing.password_symbols_half/full` を変更すると、パスワードに含めたい記号の種類を調整できます。
- `unzip_processing.archive_extensions` や `success_dir_name` などで、展開対象や保存フォルダ名を調整できます。
- `pdf_processing.auto_engine_order` を編集すると、自動選択で試すライブラリの順序を変更できます。
- ボタンのモジュールを変える場合は `app.buttons[].module` と `entry_point` を編集します。
